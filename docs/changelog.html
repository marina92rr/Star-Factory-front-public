pending 24 de febrero de 2026 fix: billing state bugs — isLoadingBill, date scope, responseType, dead code

Que cambia

  • isLoadingBill en productClientSlice pasa de [] (array vacío) a false (booleano), y se añade reducer onStartLoadingBill.
  • startDeleteBillClient ahora recibe date como parámetro — antes usaba una variable date que no existía en scope, causando ReferenceError.
  • Se añade estado de carga "Cargando facturas..." en PageDailyBill mientras se fetching facturas.
  • responseType: "blob" ahora es condicional: solo se usa cuando la acción requiere descarga de PDF (save/save_send en crear, print/print_send en obtener).
  • Se elimina src/helpers/exportBillPdf.js (código muerto, no importado en ningún sitio).
  • Se actualiza colSpan de 7 a 8 en la fila vacía de la tabla para alinearse con las columnas reales.

Por que

  • isLoadingBill como array no funcionaba como flag booleano — impedía mostrar estado de carga correctamente.
  • La variable date en startDeleteBillClient no estaba en scope, produciendo un error silencioso al recargar facturas tras borrar.
  • Sin responseType condicional, las respuestas JSON del backend se parseaban como blob, impidiendo leer ok/bill del JSON.
  • exportBillPdf.js era dead code que añadía confusión al proyecto.

Antes / Despues

- isLoadingBill: [] // array, no sirve como flag booleano
↓
+ isLoadingBill: false // booleano correcto + reducer onStartLoadingBill
- startDeleteBillClient(id) { ... getBills(date) } // date no existe en scope
↓
+ startDeleteBillClient(id, date) { ... getBills(date) } // date recibida por parametro
- responseType: "blob" // siempre blob, rompe respuestas JSON
↓
+ responseType: needsPdf ? "blob" : undefined // condicional segun accion
